Prototype (Прототип)

Также известен как Клон
Сложность ★★☆☆☆
Популярность ★★★☆☆

Назначение

Позволяет копировать объекты, не вдаваясь в подробности их реализации, путём делегирования операции клонирования самим копируемым объектам.

Проблема

У вас есть сложный конфигурационный объект, который нужно использовать как шаблон для создания множества похожих объектов с небольшими отличиями. Создание каждого объекта с нуля через конструктор требует повторения одних и тех же шагов.

Решение

Определяется общий интерфейс с методом clone(). Каждый класс реализует клонирование, создавая копию своего объекта. В Dart можно использовать встроенный оператор каскада или реализовать метод clone() вручную.

Пример на Dart

class AppConfig {
  String apiUrl;
  int timeout;
  bool retryOnError;
  Map headers;

  AppConfig({
    required this.apiUrl,
    this.timeout = 30,
    this.retryOnError = true,
    Map? headers,
  }) : headers = headers ?? {};

  AppConfig clone() {
    return AppConfig(
      apiUrl: apiUrl,
      timeout: timeout,
      retryOnError: retryOnError,
      headers: Map.from(headers),
    );
  }
}

class Shape {
  int x, y;
  String color;

  Shape(this.x, this.y, this.color);

  Shape clone() => Shape(x, y, color);
}

class Circle extends Shape {
  int radius;

  Circle(int x, int y, String color, this.radius) : super(x, y, color);

  @override
  Circle clone() => Circle(x, y, color, radius);
}

void main() {
  final baseConfig = AppConfig(
    apiUrl: 'https://api.example.com',
    timeout: 30,
    retryOnError: true,
    headers: {'Content-Type': 'application/json'},
  );

  final devConfig = baseConfig.clone();
  devConfig.apiUrl = 'https://dev.api.example.com';
  devConfig.timeout = 60;

  final circle = Circle(10, 20, 'red', 15);
  final clonedCircle = circle.clone();
  print(clonedCircle.radius); // 15
}

Когда применять

  • Когда копирование объекта эффективнее создания нового через конструктор
  • Когда нужно избежать множества подклассов фабрики
  • Когда объекты имеют множество общих полей с небольшими отличиями

Плюсы

  • Позволяет клонировать объекты без привязки к их конкретным классам
  • Может быть альтернативой наследованию при создании объектов
  • Позволяет создавать новые объекты динамически

Минусы

  • Сложно клонировать объекты с циклическими ссылками
  • Требует внимания при глубоком vs поверхностном копировании

Связанные паттерны

  • Abstract Factory — может хранить прототипы для клонирования
  • Decorator — используется для динамического расширения клонов
