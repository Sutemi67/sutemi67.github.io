Builder (Строитель)

Также известен как Строитель
Сложность ★★☆☆☆
Популярность ★★★★☆

Назначение

Отделяет конструирование сложного объекта от его представления, позволяя создавать разные представления с помощью одного и того же процесса построения.

Проблема

Класс HTTP-запроса может иметь множество необязательных параметров: заголовки, тело, параметры запроса, таймаут, авторизацию. Использование конструктора с десятком параметров неудобно и трудночитаемо. Решение — пошаговое построение объекта.

Решение

Builder позволяет конструировать объекты шаг за шагом, вызывая методы, каждый из которых устанавливает определённую часть продукта. Процесс построения можно сделать пошаговым и возвращать сам объект строителя для цепочки вызовов.

Пример на Dart

class HttpRequest {
  final String url;
  final String method;
  final Map headers;
  final Map queryParams;
  final String? body;
  final int timeout;

  HttpRequest._({
    required this.url,
    required this.method,
    required this.headers,
    required this.queryParams,
    this.body,
    required this.timeout,
  });
}

class HttpRequestBuilder {
  String _url;
  String _method = 'GET';
  final Map _headers = {};
  final Map _queryParams = {};
  String? _body;
  int _timeout = 30;

  HttpRequestBuilder(this._url);

  HttpRequestBuilder method(String method) {
    _method = method;
    return this;
  }

  HttpRequestBuilder header(String key, String value) {
    _headers[key] = value;
    return this;
  }

  HttpRequestBuilder queryParam(String key, String value) {
    _queryParams[key] = value;
    return this;
  }

  HttpRequestBuilder body(String body) {
    _body = body;
    return this;
  }

  HttpRequestBuilder timeout(int seconds) {
    _timeout = seconds;
    return this;
  }

  HttpRequest build() {
    return HttpRequest._(
      url: _url,
      method: _method,
      headers: Map.from(_headers),
      queryParams: Map.from(_queryParams),
      body: _body,
      timeout: _timeout,
    );
  }
}

void main() {
  final request = HttpRequestBuilder('https://api.example.com/users')
    .method('POST')
    .header('Content-Type', 'application/json')
    .header('Authorization', 'Bearer token123')
    .body('{"name": "John"}')
    .timeout(10)
    .build();
}

Когда применять

  • Когда объект имеет много необязательных параметров
  • Когда процесс создания состоит из нескольких шагов
  • Когда нужно создавать разные представления одного объекта

Плюсы

  • Позволяет создавать объекты пошагово
  • Переиспользует код построения для разных представлений
  • Изолирует сложную логику конструирования

Минусы

  • Усложняет код из-за необходимости создавать отдельный класс строителя
  • Требует иммутабельности продукта

Связанные паттерны

  • Factory Method — может использоваться внутри строителя
  • Decorator — похож на цепочку вызовов, но с другой целью
