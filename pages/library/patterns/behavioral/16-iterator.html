Iterator (Итератор)

Также известен как Cursor
Сложность ★☆☆☆☆
Популярность ★★★★★

Назначение

Даёт возможность последовательно обходить элементы составных объектов, не раскрывая их внутреннее представление.

Проблема

У вас есть коллекция (дерево, граф, кастомная структура), и нужно предоставить способ последовательного обхода её элементов без раскрытия внутренней структуры. При изменении способа хранения данных придётся менять весь код, который их обходит.

Решение

Создаётся класс-итератор, который инкапсулирует логику обхода. Коллекция возвращает итератор, а клиент использует его методы (hasNext/next) для обхода. В Dart для этого можно использовать Iterable и Iterator.

Пример на Dart

class TreeNode {
  T value;
  List> children = [];

  TreeNode(this.value);

  void add(TreeNode child) => children.add(child);
}

class DepthFirstIterator implements Iterator {
  final List> _stack = [];
  TreeNode? _current;

  DepthFirstIterator(TreeNode root) {
    _stack.add(root);
  }

  @override
  T get current => _current!.value;

  @override
  bool moveNext() {
    if (_stack.isEmpty) return false;
    _current = _stack.removeLast();
    for (final child in _current!.children.reversed) {
      _stack.add(child);
    }
    return true;
  }
}

class TreeIterable extends Iterable {
  final TreeNode root;
  TreeIterable(this.root);

  @override
  Iterator get iterator => DepthFirstIterator(root);
}

void main() {
  final root = TreeNode(1);
  final child1 = TreeNode(2);
  final child2 = TreeNode(3);
  root.add(child1);
  root.add(child2);
  child1.add(TreeNode(4));
  child1.add(TreeNode(5));

  for (final value in TreeIterable(root)) {
    print(value); // 1, 2, 4, 5, 3
  }

  final iter = DepthFirstIterator(root);
  while (iter.moveNext()) {
    print(iter.current);
  }
}

Когда применять

  • Когда коллекция имеет сложную внутреннюю структуру
  • Когда нужно несколько способов обхода одной коллекции
  • Когда нужно единообразно обходить разные типы коллекций

Плюсы

  • Скрывает внутреннюю структуру коллекции
  • Позволяет реализовать разные стратегии обхода
  • Упрощает API коллекций

Минусы

  • Избыточен для простых коллекций (списков, массивов)
  • Создание итератора может быть затратным

Связанные паттерны

  • Composite — итераторы часто используются для обхода деревьев Composite
  • Factory Method — может создавать специфические итераторы
