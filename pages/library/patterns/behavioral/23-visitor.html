Visitor (Посетитель)

Также известен как Visitor
Сложность ★★★☆☆
Популярность ★★★☆☆

Назначение

Позволяет добавлять новые операции к объектам, не изменяя их классы, вынося операции в отдельные объекты-посетители.

Проблема

Есть дерево AST (абстрактного синтаксического дерева) компилятора с разными типами узлов: числа, переменные, бинарные операции. Нужно реализовать несколько операций: вывод кода, оптимизация, подсчёт выражений. Добавление каждой новой операции требует изменения всех классов узлов.

Решение

Каждый класс узла реализует метод accept(visitor), который вызывает соответствующий метод посетителя. Посетитель содержит метод для каждого типа узла. Добавление новой операции — создание нового посетителя без изменения узлов.

Пример на Dart

abstract class Expression {
  void accept(ExpressionVisitor visitor);
}

class NumberExpr implements Expression {
  final int value;
  NumberExpr(this.value);

  @override
  void accept(ExpressionVisitor visitor) => visitor.visitNumber(this);
}

class VariableExpr implements Expression {
  final String name;
  VariableExpr(this.name);

  @override
  void accept(ExpressionVisitor visitor) => visitor.visitVariable(this);
}

class AddExpr implements Expression {
  final Expression left, right;
  AddExpr(this.left, this.right);

  @override
  void accept(ExpressionVisitor visitor) => visitor.visitAdd(this);
}

abstract class ExpressionVisitor {
  void visitNumber(NumberExpr expr);
  void visitVariable(VariableExpr expr);
  void visitAdd(AddExpr expr);
}

class EvalVisitor implements ExpressionVisitor {
  final Map context;

  EvalVisitor(this.context);

  int? result;

  @override
  void visitNumber(NumberExpr expr) => result = expr.value;

  @override
  void visitVariable(VariableExpr expr) => result = context[expr.name];

  @override
  void visitAdd(AddExpr expr) {
    expr.left.accept(this);
    final leftVal = result!;
    expr.right.accept(this);
    final rightVal = result!;
    result = leftVal + rightVal;
  }
}

class PrintVisitor implements ExpressionVisitor {
  String? result;

  @override
  void visitNumber(NumberExpr expr) => result = '${expr.value}';

  @override
  void visitVariable(VariableExpr expr) => result = expr.name;

  @override
  void visitAdd(AddExpr expr) {
    expr.left.accept(this);
    final leftStr = result!;
    expr.right.accept(this);
    final rightStr = result!;
    result = '($leftStr + $rightStr)';
  }
}

void main() {
  final expr = AddExpr(NumberExpr(5), VariableExpr('x'));
  final context = {'x': 10};

  final eval = EvalVisitor(context);
  expr.accept(eval);
  print(eval.result); // 15

  final printer = PrintVisitor();
  expr.accept(printer);
  print(printer.result); // (5 + x)
}

Когда применять

  • Когда нужно выполнить операции над всеми элементами сложной структуры
  • Когда классы объектов редко меняются, но часто добавляются новые операции
  • Когда нужно собрать связанные операции в одном месте

Плюсы

  • Новые операции добавляются без изменения существующих классов
  • Связанные операции собраны в одном месте
  • Накапливает состояние при обходе структуры

Минусы

  • Добавление нового типа элемента требует изменения всех посетителей
  • Может нарушать инкапсуляцию элементов

Связанные паттерны

  • Iterator — обходит структуру; Visitor выполняет операции над элементами
  • Composite — Visitor часто применяется для обхода деревьев Composite
