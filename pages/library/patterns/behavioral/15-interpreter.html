Interpreter (Интерпретатор)

Также известен как Interpreter
Сложность ★★★☆☆
Популярность ★★☆☆☆

Назначение

Для заданного языка определяет представление его грамматики и интерпретатор предложений этого языка.

Проблема

Нужно реализовать простой калькулятор, который разбирает и вычисляет арифметические выражения, например «5 + 3 * 2». Прямой разбор строки с числами и операторами требует сложной логики, которую сложно расширять.

Решение

Определяется грамматика выражений и для каждого правила создаётся класс. AST строится из объектов этих классов. Каждый класс реализует метод interpret(), возвращающий результат вычисления для своей подветки.

Пример на Dart

abstract class Expression {
  int interpret(Map context);
}

class NumberExpression implements Expression {
  final int value;
  NumberExpression(this.value);

  @override
  int interpret(Map context) => value;
}

class VariableExpression implements Expression {
  final String name;
  VariableExpression(this.name);

  @override
  int interpret(Map context) => context[name] ?? 0;
}

class AddExpression implements Expression {
  final Expression left, right;
  AddExpression(this.left, this.right);

  @override
  int interpret(Map context) =>
      left.interpret(context) + right.interpret(context);
}

class SubtractExpression implements Expression {
  final Expression left, right;
  SubtractExpression(this.left, this.right);

  @override
  int interpret(Map context) =>
      left.interpret(context) - right.interpret(context);
}

class MultiplyExpression implements Expression {
  final Expression left, right;
  MultiplyExpression(this.left, this.right);

  @override
  int interpret(Map context) =>
      left.interpret(context) * right.interpret(context);
}

void main() {
  final expression = AddExpression(
    MultiplyExpression(NumberExpression(5), NumberExpression(3)),
    NumberExpression(2),
  );
  print(expression.interpret({})); // 17

  final context = {'x': 10, 'y': 5};
  final withVars = AddExpression(VariableExpression('x'), VariableExpression('y'));
  print(withVars.interpret(context)); // 15
}

Когда применять

  • Когда грамматика языка относительно проста и стабильна
  • Когда нужно разбирать и вычислять DSL-выражения
  • Когда эффективность не является главным приоритетом

Плюсы

  • Простота реализации и расширения грамматики
  • Хорошо подходит для DSL (domain-specific languages)

Минусы

  • Сложные грамматики приводят к большому числу классов
  • Низкая производительность для сложных выражений
