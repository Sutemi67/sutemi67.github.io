Strategy (Стратегия)

Также известен как Policy
Сложность ★★☆☆☆
Популярность ★★★★★

Назначение

Определяет семейство взаимозаменяемых алгоритмов и позволяет подставлять их в контекст, не изменяя клиентский код.

Проблема

Приложение должно поддерживать разные алгоритмы сортировки (пузырьковая, быстрая, слиянием) или разные стратегии валидации данных. Код должен легко переключаться между алгоритмами без громоздких условных конструкций.

Решение

Каждый алгоритм выносится в отдельный класс, реализующий общий интерфейс. Контекст хранит ссылку на текущую стратегию и делегирует ей выполнение. Стратегию можно менять во время выполнения.

Пример на Dart

abstract class SortStrategy {
  List sort(List data);
}

class BubbleSort implements SortStrategy {
  @override
  List sort(List data) {
    final list = List.from(data);
    for (var i = 0; i < list.length; i++) {
      for (var j = 0; j < list.length - i - 1; j++) {
        if (list[j] > list[j + 1]) {
          final temp = list[j];
          list[j] = list[j + 1];
          list[j + 1] = temp;
        }
      }
    }
    return list;
  }
}

class QuickSort implements SortStrategy {
  @override
  List sort(List data) {
    if (data.length <= 1) return data;
    final pivot = data[data.length ~/ 2];
    final less = data.where((x) => x < pivot).toList();
    final equal = data.where((x) => x == pivot).toList();
    final greater = data.where((x) => x > pivot).toList();
    return [...sort(less), ...equal, ...sort(greater)];
  }
}

class Sorter {
  SortStrategy strategy;

  Sorter(this.strategy);

  void setStrategy(SortStrategy strategy) => this.strategy = strategy;

  List sort(List data) => strategy.sort(data);
}

void main() {
  final data = [5, 3, 8, 1, 9, 2];
  final sorter = Sorter(BubbleSort());
  print(sorter.sort(data));

  sorter.setStrategy(QuickSort());
  print(sorter.sort(data));
}

Когда применять

  • Когда есть несколько вариантов алгоритма, которые могут меняться
  • Когда нужно избежать условных операторов для выбора алгоритма
  • Когда алгоритмы должны быть взаимозаменяемыми

Плюсы

  • Алгоритмы можно менять на лету
  • Изолирует код алгоритмов от клиентского кода
  • Позволяет легко добавлять новые алгоритмы

Минусы

  • Клиент должен знать, какую стратегию выбрать
  • Увеличивает количество классов

Связанные паттерны

  • State — похожая структура, но State меняет поведение при смене состояния, а Strategy — при смене алгоритма
  • Template Method — задаёт скелет алгоритма, Strategy — заменяет целый алгоритм
