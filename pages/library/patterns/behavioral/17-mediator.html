Mediator (Посредник)

Также известен как Controller, Mediator
Сложность ★★☆☆☆
Популярность ★★★★☆

Назначение

Уменьшает связанность множества взаимодействующих классов, заставляя их общаться через объект-посредник, вместо прямой связи друг с другом.

Проблема

В чат-комнате каждый пользователь может отправлять сообщения всем остальным. Если каждый пользователь будет знать о каждом, то при добавлении нового пользователя придётся обновлять всех. Связность становится слишком высокой.

Решение

Создаётся посредник (ChatRoom), который управляет коммуникацией. Пользователи отправляют сообщения посреднику, а он рассылает их остальным. Пользователи не знают друг о друге — они знают только посредника.

Пример на Dart

abstract class User {
  final String name;
  ChatRoom? mediator;

  User(this.name);

  void send(String message);
  void receive(String message, String from);
}

class ChatUser extends User {
  ChatUser(String name) : super(name);

  @override
  void send(String message) {
    mediator?.sendMessage(this, message);
  }

  @override
  void receive(String message, String from) {
    print('[$name] получил от [$from]: $message');
  }
}

class ChatRoom {
  final List _users = [];

  void addUser(User user) {
    _users.add(user);
    user.mediator = this;
  }

  void sendMessage(User sender, String message) {
    for (final user in _users) {
      if (user != sender) {
        user.receive(message, sender.name);
      }
    }
  }
}

void main() {
  final chat = ChatRoom();
  final alice = ChatUser('Alice');
  final bob = ChatUser('Bob');
  final charlie = ChatUser('Charlie');

  chat.addUser(alice);
  chat.addUser(bob);
  chat.addUser(charlie);

  alice.send('Привет всем!');
  bob.send('Привет, Alice!');
}

Когда применять

  • Когда множество объектов взаимодействуют сложным образом
  • Когда хочется уменьшить связанность между компонентами
  • Когда взаимодействие между объектами можно централизовать

Плюсы

  • Уменьшает связанность между компонентами
  • Централизует управление коммуникацией
  • Упрощает поддержку и изменение взаимодействий

Минусы

  • Посредник может стать «божественным объектом»
  • Снижает производительность из-за дополнительного уровня

Связанные паттерны

  • Observer — посредник может использовать Observer для уведомлений
  • Chain of Responsibility — передаёт по цепочке, Mediator управляет централизованно
