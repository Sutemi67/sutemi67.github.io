Memento (Снимок)

Также известен как Snapshot
Сложность ★★☆☆☆
Популярность ★★★☆☆

Назначение

Позволяет сохранять и восстанавливать состояние объекта, не раскрывая его внутреннюю структуру.

Проблема

В игре есть система чекпоинтов: игрок может сохранить прогресс и вернуться к нему позже. Нужно сохранять состояние игровых объектов, но при этом не нарушать инкапсуляцию и не раскрывать внутренние поля.

Решение

Объект-создатель (originator) создаёт снимок (memento) своего состояния. Снимок хранит состояние в неизменяемом виде, доступном только создателю. Опекун (caretaker) хранит снимки и может восстановить состояние, передав снимок обратно создателю.

Пример на Dart

class GameState {
  int health;
  int level;
  String position;

  GameState(this.health, this.level, this.position);

  GameMemento save() => GameMemento._(health, level, position);

  void restore(GameMemento memento) {
    health = memento._health;
    level = memento._level;
    position = memento._position;
  }

  @override
  String toString() => 'HP: $health, Level: $level, Pos: $position';
}

class GameMemento {
  final int _health;
  final int _level;
  final String _position;

  GameMemento._(this._health, this._level, this._position);
}

class SaveManager {
  final List _history = [];

  void save(GameState state) => _history.add(state.save());

  void load(GameState state, int index) {
    if (index < _history.length) {
      state.restore(_history[index]);
    }
  }
}

void main() {
  final game = GameState(100, 1, 'start');
  final saves = SaveManager();

  game.health = 80;
  game.level = 2;
  game.position = 'forest';
  saves.save(game);
  print(game); // HP: 80, Level: 2, Pos: forest

  game.health = 30;
  game.position = 'boss';
  print(game); // HP: 30, Level: 2, Pos: boss

  saves.load(game, 0);
  print(game); // HP: 80, Level: 2, Pos: forest
}

Когда применять

  • Когда нужно сохранять и восстанавливать состояние объекта
  • Когда нужна поддержка отмены операций (undo) через сохранение состояний
  • Когда прямое сохранение состояния нарушает инкапсуляцию

Плюсы

  • Не нарушает инкапсуляцию исходного объекта
  • Упрощает реализацию отмены операций
  • Позволяет хранить историю изменений

Минусы

  • Требует много памяти при частых сохранениях
  • Усложняет код из-за дополнительного класса снимка

Связанные паттерны

  • Command — может использовать Memento для реализации undo
  • Iterator — может обходить историю снимков
