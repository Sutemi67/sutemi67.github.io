Command (Команда)

Также известен как Action, Transaction
Сложность ★★☆☆☆
Популярность ★★★★☆

Назначение

Превращает запросы в объекты, позволяя передавать их как аргументы, ставить в очередь, логировать, а также поддерживать отмену операций (undo).

Проблема

В текстовом редакторе есть операции: вставка текста, удаление, копирование, вставка. Нужно реализовать отмену (undo/redo) и историю действий. Если каждая операция напрямую вызывает методы редактора, сложно отслеживать историю.

Решение

Каждая операция представляется отдельным объектом-командой с методами execute() и undo(). Команды хранятся в истории. Для отмены вызывается undo() последней команды. Команды можно ставить в очередь, логировать, передавать как параметры.

Пример на Dart

abstract class Command {
  void execute();
  void undo();
}

class TextEditor {
  String text = '';

  void insert(String at, String content) {
    final pos = text.indexOf(at);
    text = text.substring(0, pos) + content + text.substring(pos);
  }

  void delete(int start, int length) {
    text = text.substring(0, start) + text.substring(start + length);
  }
}

class InsertCommand implements Command {
  final TextEditor editor;
  final String at;
  final String content;
  late int _insertPos;

  InsertCommand(this.editor, this.at, this.content);

  @override
  void execute() {
    _insertPos = editor.text.indexOf(at);
    editor.insert(at, content);
  }

  @override
  void undo() {
    editor.delete(_insertPos, content.length);
  }
}

class CommandHistory {
  final List _history = [];
  int _current = -1;

  void execute(Command command) {
    command.execute();
    _history.add(command);
    _current++;
  }

  void undo() {
    if (_current >= 0) {
      _history[_current].undo();
      _current--;
    }
  }
}

void main() {
  final editor = TextEditor();
  editor.text = 'Hello World!';
  final history = CommandHistory();

  final cmd = InsertCommand(editor, 'World', ' beautiful');
  history.execute(cmd);
  print(editor.text); // Hello beautiful World!

  history.undo();
  print(editor.text); // Hello World!
}

Когда применять

  • Когда нужна отмена (undo/redo) операций
  • Когда нужно ставить операции в очередь или логировать их
  • Когда нужно параметризовать объекты выполняемыми действиями

Плюсы

  • Отделяет отправителя от получателя
  • Поддержка undo/redo «из коробки»
  • Возможность составлять сложные команды из простых

Минусы

  • Множество мелких классов команд
  • Усложнение кода для простых операций

Связанные паттерны

  • Memento — альтернативный способ реализации undo
  • Observer — может уведомлять об изменении состояния после выполнения команды
