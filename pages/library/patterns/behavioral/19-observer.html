Поведенческий

Observer (Наблюдатель)

Также известен как Listener, Publisher-Subscriber
Сложность ★★☆☆☆
Популярность ★★★★★

Назначение

Создаёт механизм подписки, позволяющий одним объектам (издателям) уведомлять другие объекты (подписчиков) об изменениях своего состояния.

Проблема

В приложении с курсом валют несколько виджетов должны обновляться при изменении курса: график, таблица, индикатор. Если жёстко связывать виджеты с источником данных, добавление нового виджета потребует изменения кода источника.

Решение

Издатель (Subject) хранит список подписчиков (Observers) и уведомляет их при изменении состояния. Подписчики реализуют общий интерфейс, а издатель не знает их конкретных типов. В Dart эта концепция реализована через StreamSubscription и ChangeNotifier.

Пример на Dart

abstract class Observer {
  void update(double rate);
}

class CurrencyExchange {
  final List _observers = [];
  double _rate = 0;

  void subscribe(Observer observer) => _observers.add(observer);
  void unsubscribe(Observer observer) => _observers.remove(observer);

  void _notify() {
    for (final observer in _observers) {
      observer.update(_rate);
    }
  }

  void setRate(double newRate) {
    _rate = newRate;
    _notify();
  }
}

class ChartWidget implements Observer {
  @override
  void update(double rate) {
    print('[Chart] Обновление графика: курс $rate');
  }
}

class TableWidget implements Observer {
  @override
  void update(double rate) {
    print('[Table] Обновление таблицы: курс $rate');
  }
}

class IndicatorWidget implements Observer {
  @override
  void update(double rate) {
    print('[Indicator] Новый курс: $rate');
  }
}

void main() {
  final exchange = CurrencyExchange();
  final chart = ChartWidget();
  final table = TableWidget();

  exchange.subscribe(chart);
  exchange.subscribe(table);
  exchange.subscribe(IndicatorWidget());

  exchange.setRate(92.45);
  exchange.setRate(93.10);
}

Когда применять

  • Когда изменения одного объекта требуют обновления других объектов
  • Когда количество подписчиков неизвестно и может меняться динамически
  • Когда нужно слабое связывание между издателями и подписчиками

Плюсы

  • Слабая связанность между издателями и подписчиками
  • Поддержка broadcast-уведомлений
  • Динамическое управление подписками

Минусы

  • Подписчики получают уведомления в случайном порядке
  • Утечки памяти при забытой отписке
  • Сложность отладки цепочек уведомлений

Связанные паттерны

  • Mediator — посредник может использовать Observer для уведомления
  • State — может уведомлять об изменении состояния через Observer
