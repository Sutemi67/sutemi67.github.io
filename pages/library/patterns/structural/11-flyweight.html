Flyweight (Легковес)

Также известен как Кэш
Сложность ★★★☆☆
Популярность ★★☆☆☆

Назначение

Позволяет вместить большее количество объектов в выделенную оперативную память за счёт разделения общего внутреннего состояния между многими объектами.

Проблема

В текстовом редакторе каждый символ представлен объектом, содержащим информацию о шрифте, размере, цвете и позиции. При открытии большого документа (100 000+ символов) создаётся огромное количество объектов, что приводит к нехватке памяти.

Решение

Разделяем состояние на внутреннее (шрифт, размер, цвет) и внешнее (позиция на экране). Внутреннее состояние выносится в отдельные объекты-легковесы и переиспользуется. Внешнее состояние остаётся в контексте и передаётся при необходимости.

Пример на Dart

class TextStyle {
  final String font;
  final int size;
  final String color;
  final bool bold;

  TextStyle(this.font, this.size, this.color, this.bold);

  @override
  bool operator ==(Object other) =>
      other is TextStyle &&
      other.font == font &&
      other.size == size &&
      other.color == color &&
      other.bold == bold;

  @override
  int get hashCode => Object.hash(font, size, color, bold);
}

class TextStyleFactory {
  static final Map _cache = {};

  static TextStyle getStyle(String font, int size, String color, bool bold) {
    final key = TextStyle(font, size, color, bold);
    _cache.putIfAbsent(key, () => key);
    return _cache[key]!;
  }
}

class Character {
  final String char;
  final TextStyle style;
  final int x, y;

  Character(this.char, this.style, this.x, this.y);

  void render() {
    print('$char (${style.font}, ${style.size}pt) at ($x, $y)');
  }
}

void main() {
  final defaultStyle = TextStyleFactory.getStyle('Arial', 12, 'black', false);
  final headingStyle = TextStyleFactory.getStyle('Arial', 18, 'black', true);
  final codeStyle = TextStyleFactory.getStyle('Courier', 10, 'green', false);

  final chars = [
    Character('H', headingStyle, 0, 0),
    Character('e', defaultStyle, 10, 0),
    Character('l', defaultStyle, 20, 0),
    Character('l', defaultStyle, 30, 0),
    Character('o', codeStyle, 40, 0),
  ];

  for (final c in chars) {
    c.render();
  }
}

Когда применять

  • Когда приложение использует большое количество однотипных объектов
  • Когда бóльшая часть состояния объектов является общей
  • Когда затраты на хранение объектов критичны для производительности

Плюсы

  • Существенно экономит память при большом количестве объектов
  • Уменьшает количество создаваемых объектов

Минусы

  • Усложняет код разделением на внутреннее и внешнее состояние
  • Требует дополнительных затрат на поиск/создание легковеса

Связанные паттерны

  • Composite — может использоваться вместе с Flyweight
  • Proxy — может управлять созданием легковесов
