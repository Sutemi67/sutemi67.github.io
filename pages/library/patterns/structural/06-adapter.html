Adapter (Адаптер)

Также известен как Wrapper
Сложность ★★☆☆☆
Популярность ★★★★★

Назначение

Преобразует интерфейс одного класса в другой интерфейс, ожидаемый клиентом. Адаптер обеспечивает совместную работу классов, невозможную в обычных условиях из-за несовместимости интерфейсов.

Проблема

Ваше приложение использует единый интерфейс для работы с данными, но нужно интегрировать стороннюю XML-библиотеку, которая имеет совершенно другой API. Переписывать библиотеку нельзя, менять интерфейс приложения — дорого.

Решение

Создаётся адаптер — класс-прослойка, который преобразует вызовы методов целевого интерфейса в вызовы методов адаптируемого класса. Клиент работает с адаптером через знакомый интерфейс, не зная о несовместимости.

Пример на Dart

// Целевой интерфейс
abstract class DataProvider {
  List> fetchData();
}

// Сторонняя библиотека (несовместимый интерфейс)
class XmlLibrary {
  String getXml() {
    return 'Item1';
  }
}

// Адаптер
class XmlAdapter implements DataProvider {
  final XmlLibrary _xmlLibrary;

  XmlAdapter(this._xmlLibrary);

  @override
  List> fetchData() {
    final xml = _xmlLibrary.getXml();
    return _parseXml(xml);
  }

  List> _parseXml(String xml) {
    final items = >[];
    final nameRegex = RegExp(r'(.*?)');
    for (final match in nameRegex.allMatches(xml)) {
      items.add({'name': match.group(1)});
    }
    return items;
  }
}

void main() {
  final provider = XmlAdapter(XmlLibrary());
  final data = provider.fetchData();
  print(data);
}

Когда применять

  • Когда нужно использовать сторонний класс с несовместимым интерфейсом
  • Когда нужно заменить одну библиотеку на другую без изменения клиентского кода
  • Когда нужно обеспечить совместимость между старой и новой частями системы

Плюсы

  • Изолирует клиента от несовместимых интерфейсов
  • Не требует изменения существующего кода
  • Позволяет интегрировать legacy-компоненты

Минусы

  • Увеличивает сложность системы дополнительными классами
  • Может снижать производительность из-за прослойки

Связанные паттерны

  • Bridge — похож по структуре, но решает другую задачу
  • Decorator — тоже обёртка, но для расширения функциональности
