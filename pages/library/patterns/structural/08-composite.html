Composite (Компоновщик)

Также известен как Дерево
Сложность ★★★☆☆
Популярность ★★★★☆

Назначение

Позволяет сгруппировать объекты в древовидную структуру и работать с отдельными объектами и их группами единообразно.

Проблема

Нужно представить файловую систему: файлы и папки. Папки могут содержать как файлы, так и другие папки. Клиентский код должен одинаково работать и с файлами, и с папками — например, вычислять общий размер или выводить содержимое.

Решение

Определяется общий интерфейс Component с методами, общими для листьев (файлов) и контейнеров (папок). Листья реализуют поведение напрямую, а контейнеры делегируют работу своим дочерним элементам.

Пример на Dart

abstract class FileSystemComponent {
  String get name;
  int getSize();
  void display(String indent);
}

class File implements FileSystemComponent {
  @override
  final String name;
  final int size;

  File(this.name, this.size);

  @override
  int getSize() => size;

  @override
  void display(String indent) {
    print('$indent $name ($size KB)');
  }
}

class Folder implements FileSystemComponent {
  @override
  final String name;
  final List children = [];

  Folder(this.name);

  void add(FileSystemComponent component) => children.add(component);
  void remove(FileSystemComponent component) => children.remove(component);

  @override
  int getSize() {
    return children.fold(0, (sum, child) => sum + child.getSize());
  }

  @override
  void display(String indent) {
    print('$indent $name/');
    for (final child in children) {
      child.display('$indent  ');
    }
  }
}

void main() {
  final root = Folder('root');
  final docs = Folder('documents');
  final pics = Folder('pictures');

  docs.add(File('resume.pdf', 200));
  docs.add(File('notes.txt', 50));
  pics.add(File('photo.jpg', 1500));
  pics.add(File('logo.png', 300));

  root.add(docs);
  root.add(pics);
  root.add(File('readme.md', 10));

  root.display('');
  print('Общий размер: ${root.getSize()} KB');
}

Когда применять

  • Когда нужно представить иерархию «часть-целое»
  • Когда клиенты должны единообразно работать с простыми и составными объектами
  • Когда структура объектов может быть рекурсивной

Плюсы

  • Упрощает клиентский код (одинаковая работа с листьями и контейнерами)
  • Позволяет легко добавлять новые типы компонентов
  • Реализует рекурсивную обработку

Минусы

  • Может сделать систему слишком обобщённой
  • Трудно наложить ограничения на типы дочерних элементов

Связанные паттерны

  • Decorator — похож по структуре, но добавляет функциональность, а не группирует
  • Iterator — часто используется для обхода дерева Composite
