Facade (Фасад)

Также известен как Facade
Сложность ★☆☆☆☆
Популярность ★★★★★

Назначение

Предоставляет простой интерфейс к сложной системе классов, библиотеке или набору подсистем, скрывая внутреннюю сложность.

Проблема

Для конвертации видеофайла приложению нужно работать с несколькими подсистемами: чтение файла, декодирование, обработка звука, кодирование, запись. Клиентский код вынужден знать все детали и порядок вызовов каждой подсистемы.

Решение

Создаётся фасад — класс, который предоставляет простой метод для выполнения сложной операции, скрывая за собой взаимодействие с подсистемами. Клиент вызывает один метод фасада, а тот за кулисами управляет всеми подсистемами.

Пример на Dart

class VideoFile {
  final String filename;
  VideoFile(this.filename);
}

class Codec {
  void decode(VideoFile file) => print('Декодирование ${file.filename}');
  void encode(VideoFile file) => print('Кодирование ${file.filename}');
}

class AudioMixer {
  void process() => print('Обработка аудиодорожки');
}

class FileWriter {
  void save(String filename) => print('Сохранение файла $filename');
}

class VideoConverter {
  final Codec _codec = Codec();
  final AudioMixer _mixer = AudioMixer();
  final FileWriter _writer = FileWriter();

  void convert(String filename, String format) {
    final file = VideoFile(filename);
    _codec.decode(file);
    _mixer.process();
    _codec.encode(file);
    _writer.save('${filename.split('.').first}.$format');
  }
}

void main() {
  final converter = VideoConverter();
  converter.convert('video.avi', 'mp4');
}

Когда применять

  • Когда нужно предоставить простой интерфейс к сложной подсистеме
  • Когда нужно уменьшить зависимость клиентского кода от подсистемы
  • Когда нужно разбить систему на слои (фасад — точка входа в каждый слой)

Плюсы

  • Изолирует клиентов от сложности подсистемы
  • Упрощает использование сложных библиотек
  • Уменьшает связанность кода

Минусы

  • Фасад может стать «божественным объектом», знающим слишком много
  • Скрывает потенциально полезную функциональность подсистемы

Связанные паттерны

  • Adapter — меняет интерфейс, Facade создаёт новый упрощённый интерфейс
  • Proxy — контролирует доступ, Facade скрывает сложность
