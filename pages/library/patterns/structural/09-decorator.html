Decorator (Декоратор)

Также известен как Wrapper
Сложность ★★☆☆☆
Популярность ★★★★★

Назначение

Динамически добавляет новые поведения объектам, оборачивая их в классы-декораторы. Является гибкой альтернативой наследованию для расширения функциональности.

Проблема

Есть класс для работы с данными, который может читать и записывать данные. Нужно добавить функциональность: сжатие, шифрование, логирование. Создание всех комбинаций через наследование приводит к взрыву подклассов.

Решение

Создаётся базовый декоратор, который содержит ссылку на оборачиваемый объект и делегирует ему вызовы. Конкретные декораторы переопределяют методы, добавляя поведение до или после делегирования.

Пример на Dart

abstract class DataSource {
  void writeData(String data);
  String readData();
}

class FileDataSource implements DataSource {
  final String filename;
  String _data = '';

  FileDataSource(this.filename);

  @override
  void writeData(String data) {
    _data = data;
    print('Записано в $filename: $data');
  }

  @override
  String readData() => _data;
}

class DataSourceDecorator implements DataSource {
  final DataSource wrappee;

  DataSourceDecorator(this.wrappee);

  @override
  void writeData(String data) => wrappee.writeData(data);
  @override
  String readData() => wrappee.readData();
}

class EncryptionDecorator extends DataSourceDecorator {
  EncryptionDecorator(DataSource wrappee) : super(wrappee);

  @override
  void writeData(String data) {
    final encrypted = _encrypt(data);
    super.writeData(encrypted);
  }

  @override
  String readData() {
    return _decrypt(super.readData());
  }

  String _encrypt(String data) => data.split('').reversed.join();
  String _decrypt(String data) => data.split('').reversed.join();
}

class CompressionDecorator extends DataSourceDecorator {
  CompressionDecorator(DataSource wrappee) : super(wrappee);

  @override
  void writeData(String data) {
    final compressed = _compress(data);
    super.writeData(compressed);
  }

  @override
  String readData() {
    return _decompress(super.readData());
  }

  String _compress(String data) => data.substring(0, data.length ~/ 2);
  String _decompress(String data) => data + data;
}

void main() {
  DataSource source = FileDataSource('test.txt');
  source = EncryptionDecorator(source);
  source = CompressionDecorator(source);

  source.writeData('Hello, World!');
  print(source.readData());
}

Когда применять

  • Когда нужно добавлять обязанности динамически и прозрачно
  • Когда невозможно расширение через наследование
  • Когда нужно избежать появления множества подклассов для каждой комбинации

Плюсы

  • Гибкая альтернатива наследованию
  • Динамическое добавление/удаление поведения
  • Комбинирование нескольких декораторов

Минусы

  • Множество мелких классов усложняет отладку
  • Порядок декораторов может влиять на результат

Связанные паттерны

  • Adapter — меняет интерфейс, Decorator расширяет поведение
  • Composite — похож по структуре, но решает другую задачу
