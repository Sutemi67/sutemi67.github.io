Bridge (Мост)

Также известен как Handle/Body
Сложность ★★★☆☆
Популярность ★★★☆☆

Назначение

Разделяет абстракцию и реализацию так, чтобы они могли изменяться независимо друг от друга.

Проблема

Есть пульт дистанционного управления, который должен работать с разными устройствами: TV, Radio, проектором. Если наследовать пульты для каждого устройства, получится взрыв классов. Изменение пульта потребует правок во всех наследниках.

Решение

Вместо наследования абстракция (Remote) хранит ссылку на реализацию (Device). Абстракция делегирует вызовы методам реализации. Теперь иерархии пультов и устройств независимы — можно добавлять новые типы пультов и устройств без изменения друг друга.

Пример на Dart

abstract class Device {
  bool isEnabled();
  void enable();
  void disable();
  int getVolume();
  void setVolume(int volume);
}

class TV implements Device {
  bool _on = false;
  int _volume = 50;

  @override
  bool isEnabled() => _on;
  @override
  void enable() { _on = true; print('TV включён'); }
  @override
  void disable() { _on = false; print('TV выключен'); }
  @override
  int getVolume() => _volume;
  @override
  void setVolume(int volume) { _volume = volume; }
}

class Radio implements Device {
  bool _on = false;
  int _volume = 30;

  @override
  bool isEnabled() => _on;
  @override
  void enable() { _on = true; print('Radio включено'); }
  @override
  void disable() { _on = false; print('Radio выключено'); }
  @override
  int getVolume() => _volume;
  @override
  void setVolume(int volume) { _volume = volume; }
}

class Remote {
  final Device device;

  Remote(this.device);

  void togglePower() {
    if (device.isEnabled()) {
      device.disable();
    } else {
      device.enable();
    }
  }

  void volumeUp() => device.setVolume(device.getVolume() + 10);
  void volumeDown() => device.setVolume(device.getVolume() - 10);
}

void main() {
  final tv = TV();
  final remote = Remote(tv);
  remote.togglePower();
  remote.volumeUp();
}

Когда применять

  • Когда нужно избежать жёсткой привязки абстракции к реализации
  • Когда абстракция и реализация должны расширяться независимо
  • Когда изменения в реализации не должны влиять на клиентский код

Плюсы

  • Независимое расширение абстракций и реализаций
  • Соблюдение принципа единственной ответственности
  • Поддержка разных платформ и устройств

Минусы

  • Усложняет код из-за дополнительных интерфейсов
  • Требует продуманного разделения на абстракцию и реализацию

Связанные паттерны

  • Adapter — делает совместимыми интерфейсы; Bridge разделяет абстракцию и реализацию
  • Composite — может использоваться вместе с Bridge
