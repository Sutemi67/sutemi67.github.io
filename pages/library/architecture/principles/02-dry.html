Принципы

DRY — Don't Repeat Yourself (Не повторяй себя)

Также известен как DIE — Duplication Is Evil
Сложность ★☆☆☆☆
Применимость ★★★★★

Назначение

Каждая часть знания должна иметь единственное непротиворечивое представление в системе. DRY призывает избегать дублирования логики, данных и знаний, вынося повторяющийся код в единое место — функции, классы или модули.

Проблема

Когда один и тот же фрагмент логики повторяется в разных местах, изменение этого поведения требует правки в каждом из них. Это приводит к ошибкам (забыли обновить одно из мест), неконсистентности поведения и усложняет поддержку кода.

Решение

Выделить повторяющуюся логику в единую абстракцию: функцию, метод, класс или модуль. Все места, которым нужна эта логика, обращаются к единому источнику. Изменение в одном месте автоматически применяется везде.

Важно: DRY касается не только кода, но и знаний. Документация, конфигурация, схемы БД — всё должно иметь единственный источник истины.

Пример на Dart

Плохо — дублирование логики суммирования:

double calculateTotalWithTax(List prices) {
  double sum = 0;
  for (final p in prices) { sum += p; }
  return sum * 1.2;
}

double calculateTotalWithDiscount(List prices) {
  double sum = 0;
  for (final p in prices) { sum += p; }
  return sum * 0.9;
}

Хорошо — единая функция суммирования:

double sum(List values) =>
    values.fold(0, (a, b) => a + b);

double calculateTotalWithTax(List prices) =>
    sum(prices) * 1.2;

double calculateTotalWithDiscount(List prices) =>
    sum(prices) * 0.9;

Плюсы

  • Снижается количество ошибок при изменениях
  • Код проще поддерживать и рефакторить
  • Поведение системы консистентно
  • Упрощается понимание кода новыми разработчиками

Минусы

  • Чрезмерное применение приводит к преждевременной абстракции
  • Избыточное обобщение усложняет код там, где дублирование было бы уместно
  • Требует опыта, чтобы отличить «случайное» дублирование от «истинного»

Связанные принципы

  • Single Responsibility — DRY помогает соблюдать SRP
  • YAGNI — баланс между DRY и YAGNI: абстракция нужна, только когда дублирование реально возникло
