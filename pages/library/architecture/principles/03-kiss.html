Принципы

KISS — Keep It Simple, Stupid (Будь проще)

Также известен как Keep It Short and Simple
Сложность ★☆☆☆☆
Применимость ★★★★★

Назначение

Большинство систем работают лучше всего, если они остаются простыми, а не усложняются. Простота должна быть ключевой целью в проектировании — излишняя сложность ведёт к ошибкам, дорогой поддержке и трудностям в развитии.

Проблема

Разработчики склонны усложнять решения: использовать паттерны там, где достаточно простой функции, строить многоуровневые абстракции для тривиальных задач, писать «на вырост». Сложный код тяжело читать, тестировать и изменять.

Решение

Начинать с самого простого рабочего варианта. Если решение кажется сложным — вероятно, вы что-то делаете не так. Задавайте себе вопрос: «Можно ли сделать это проще?». Простой код легче понять, протестировать и изменить.

KISS не означает «примитивно». Это означает «настолько просто, насколько возможно, но не проще».

Пример на Dart

Плохо — излишняя сложность для простой задачи:

class StringReverser {
  String reverse(String input) {
    return input.split('').fold('', (prev, c) => c + prev);
  }
}

Хорошо — простое и понятное решение:

String reverse(String input) => input.split('').reversed.join();

Плюсы

  • Код легче читать и понимать
  • Меньше шансов допустить ошибку
  • Быстрее разрабатывать и поддерживать
  • Проще тестировать и отлаживать

Минусы

  • Чрезмерное упрощение может игнорировать необходимую гибкость
  • Не всегда очевидно, где проходит граница «достаточной простоты»
  • Простота в коде иногда требует более сложных размышлений при проектировании

Связанные принципы

  • YAGNI — не усложняй решение тем, что не нужно прямо сейчас
  • DRY — абстракция должна быть простой и очевидной
